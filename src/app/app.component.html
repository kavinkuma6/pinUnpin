Challenge Overview

The app is browser based web app which needs to create/update a reusable UI components. No codebase provided. You can create a simple demo app using a button to implement the pin-unpin window. Goal of this challenge is to implement reusable component for pin-unpin window.

General Requirements

Technologies and Frameworks

  • Angular 8+
  • HTML5 & CSS3
  • Angular Material

Validation

  • Please use a linter like ESLint
  • Usage of any third-party angular library is subject to approval and should be comply with open source license

Verification

  • A Readme file which contains how to use the components
  • A simple demo app to implement the pin-unpin window components

Layout Specification

  • Browser based web app for desktop/laptop all resolution

Platform Requirement

  • Latest Firefox, Latest Safari & Chrome Browsers (Mac & Windows)

Individual Requirements

Pin-Unpin Window Dialog Develop a reusable pin-unpin dialog window component.

  • Dialog window based on Angular Material
  • Dialog window should have the feature for pin and unpin
  • When the dialog window on the pin state: it will be shown in a particular position within the whole screen
  • When the dialog window on the unpin state: it will be a modal dialog that can be moved anywhere within the whole screen

Framework Specific Requirements

  • Code should follow the best practices of ECMAScript 5 (ES5) or above.
  • Split the code into Reusable components.
  • For AngularJS based applications these are the expected modules: Core, Shared and lazy-loaded Feature modules.
    • Core Modules: All services which have to have one and only one instance per application (singleton services) should be implemented here.
    • SharedModule: All the “dumb” components and pipes should be implemented here.
    • Lazy Loading: We should lazy load our feature modules whenever possible.
  • If the official framework documentation says a specific way or approach to do something, and you did differently with no solid reason, that's valid for scoring down.

Licenses & Attribution

  • Third-party assets used to build your item must be properly licensed or free for commercial use. MIT, some modified BSD, Apache 2 licenses are ok. If a library is not commercial friendly you will need to get our approval first.
  • Sufficient information regarding third-party assets must be present in your documentation. This includes the author, license info and a direct link to the asset online.

Deliverables

  • Zip file with simple demo Angular application
  • A Readme file how to run and verify your solutions
